  1. Introduo

    1. Objetivo:

      1. LoopBack 4.0 um framework Node.js escrito na linguagem TypeScript, altamente extensvel e de cdigo aberto baseado no Express. Ele permite que voc crie rapidamente APIs e micros servios compostos de sistemas de back-end, como bancos de dados e servios SOAP ou REST.

    2. Pre-requisitos:

      1. O programa nodejs um programa tipo browser que executa JavaScript fora de um navegador web, permitindo acesso a funes do sistema operacional no lado do servidor.

      2. O programa npm significa Node Package Manager e uma ferramenta que nos permite instalar e gerenciar pacotes de nodejs como dependncias.

      3. O programa npx um executor de pacote NPM que torna realmente fcil instalar qualquer tipo de executvel de nodejs que normalmente teria sido instalado usando NPM.

      2. Integra com bancos de dados, servios web e outras plataformas usando conectores.

        1. Os conectores so uma espcie de api para acessar as seguintes tecnologias:
          1. Conectores de acesso a memria;

  2. Descrio

    1. Viso geral.

      1. Veja o conceito do loopback 4 para ter uma viso geral do framework.

        1. Application : A classe central para configurar todos os componentes, controladores, servidores e ligaes do seu mdulo. A classe Application estende Context e fornece os controles para iniciar e parar seus servidores associados.

        2. Server : Classe responsvel pela implementao para transportes / protocolos de entrada, como REST (http, https), gRPC (http2) e graphQL (http, https). Ele geralmente escuta em um endpoint (protocol/host/port), trata das solicitaes de entrada e retorna as respostas apropriadas.

        3. Controller : Classe que implementa operaes definidas pela API REST do aplicativo. Ele implementa a lgica de negcios de um aplicativo e atua como uma ponte entre a API HTTP/REST e os modelos (models) de domnio/database. Um controlador opera apenas em entradas processadas e abstraes de servios de back-end /databases.

        4. Interceptor : uma funo que intercepta invocaes de mtodos estticos ou de instncia em uma classe ou objeto.

        5. Route : o mapeamento entre sua especificao de API e uma operao. Ele diz ao LoopBack para qual Operao invoke() quando recebe uma solicitao HTTP.

        6. Sequence : um agrupamento sem estado de aes que controlam como um servidor responde s solicitaes.

        7. DataSource : um nome dado configurao de conexo para um banco de dados de um servidor. O nome normalmente utilizado quando cria-se uma consulta para o banco de dados. O DSN (Datasource Name) no tem que ser o mesmo que o nome do arquivo para o banco de dados. Por exemplo, um arquivo de banco de dados chamado "amigos.mdb" poderia ser definido com um DSN de "escola". O DSN "escola" seria ento utilizado para referir-se ao banco de dados quando realizasse uma consulta.

        8. Model : So classes com a definio dos campos da fonte de dados. O @loopback/repository mdulo fornece decoradores especiais para adicionar metadados s classes TypeScript/JavaScript para us-los com o DataSource. Alm disso, o@loopback/repository-json-schema mdulo usa os metadados dos decoradores para construir um esquema JSON correspondente.

        9. Repository : um tipo de servio que representa uma coleo de dados em um DataSource.

        10. Relation : Um mapeamento entre dois modelos que descreve um link do mundo real entre eles e expe APIs CRUD com base na configurao.

        11. Decorator : o padro usado para anotar ou modificar suas declaraes de classe e seus membros com metadados.

      2. Aqui esto as infraestruturas que fazem todos os artefatos trabalharem juntos:

        1. Context : Uma abstrao de estados e dependncias em seu aplicativo que o LoopBack usa para gerenciar tudo. um registro global para tudo em seu aplicativo (configuraes, estado, dependncias, classes e assim por diante).

        2. Binding : Uma abstrao de itens gerenciados por um contexto. Cada ligao possui uma chave exclusiva dentro do contexto e um provedor de valor para resolver a chave para um valor.

        3. Dependency injection : A tcnica usada para separar a construo de dependncias de uma classe ou funo de seu comportamento para manter o cdigo fracamente acoplado.

        4. Component : um pacote que rene uma ou mais extenses LoopBack.

      1. Nome da classe: PascalCase.

        1. PascalCase uma conveno de nomenclatura em que a primeira letra de cada palavra em uma palavra composta maiscula. Veja mais...

          1. Exemplo:

               export class CadastroDeClientes extends Entity {}

      2. O nome do arquivo: kebab-case.

        1. kebab-case - uma conveno de nomenclatura de varivel de programao onde um desenvolvedor substitui os espaos entre as palavras por um travesso e todas as letras devem ser minusculas ou maisculas. Veja mais...
          1. Exemplo:
            1. cadastro-de-clientes.ts
            2. CADASTRO-DE-CLIENTES.TS

            My-Model O nome da classe no pode conter alguns smbolos Invlido Invlido

    3. lb4 - Aplicativo de linha de comando:

      1. Usage:

        1. lb4 app [<name>] [options]

      2. Options:

        1. -h, --help # Imprimir as opes e o uso do gerador
        2. --skip-cache # No lembrar respostas do prompt Default:false
        3. --skip-install # No instalar dependncias automaticamente Default: false
        4. --force-install # Erro de falha na instalao de dependncias Default: false
        5. --ask-answered # Show prompts for already configured options Default: false
        6. --applicationName # Nome da classe do aplicativo
        7. --docker # Incluir Dockerfile e .dockerignore
        8. --repositories # Inclua importaes de repositrio e RepositoryMixin
        9. --services # Inclua importaes de proxy de servio e ServiceMixin
        10. --apiconnect # Incluir ApiConnectComponent
        11. --description # Descrio para o application
        12. --outdir # Diretrio-raiz do projeto para o application
        13. --eslint # Ativar eslint
        14. --prettier # Ativar o Prettier
        15. --mocha # Ativar mocha
        16. --loopbackBuild # Use @loopback/build
        17. --vscode # Usar definies de VSCode pr-configuradas
        18. --private # Marque o projeto como privado (excludo da publicao npm)
        19. -c,--config # Nome ou valor do arquivo JSON para configurar opes
        20. -y,--yes # Ignore todos os prompts de confirmao com valor padro ou fornecido
        21. --format # Formate o cdigo gerado usando npm run lint:fix
        22. -pm, --packageManager # Mude o gerenciador de pacote padro

      3. Arguments:

        1. name # Nome do projeto para o application Type: String Required: false

      4. Available commands:

        1. lb4 app # Cria um novo aplicativo LoopBack4 usando a API REST.

        2. lb4 extension # Cria uma nova extenso LoopBack4.

        3. lb4 controller # Adicionar um novo controlador a um aplicativo LoopBack 4.

        4. lb4 datasource # Adicionar uma nova fonte de dados a um aplicativo LoopBack 4.

        5. lb4 import-lb3-models # Importe um ou mais modelos LoopBack 3 para um aplicativo LoopBack 4.

        6. lb4 model # Adicionar um novo modelo a um aplicativo LoopBack 4.

        7. lb4 repository # Adicionar novos repositrios para o (s) modelo (s) selecionado (s) a um aplicativo LoopBack 4.

        8. lb4 service # Adicionar um novo servio remoto ou local a um aplicativo LoopBack 4.

        9. lb4 example # LoopBack 4 vem com vrios projetos de exemplo.

        10. lb4 openapi # Gerar controladores e modelos a partir de especificaes OpenAPI.

        11. lb4 observer # Gerar observadores de ciclo de vida para iniciar / parar o aplicativo.

        12. lb4 interceptor # Gerar interceptores.

        13. lb4 discover # Descubra modelos de bancos de dados relacionais.

        14. lb4 relation # Adicione uma relao entre dois modelos em um aplicativo LoopBack 4.

        15. lb4 update # O lb4 update comando executado dentro de um projeto LoopBack 4 e verifica as dependncias em relao verso atual do @loopback/cli. Por padro, ele verifica a correspondncia exata. Use a --semver opo para verificar a compatibilidade usando semver semntica.

        16. lb4 rest-crud # Gerar configuraes restantes para endpoints do modelo.

        17. lb4 copyright # O lb4 copyright comando executado dentro de um projeto Node.js package.json para adicionar ou atualizar o cabealho de copyright / licena para arquivos JavaScript e TypeScript com base em um package.json histrico git.
          O comando tambm oferece suporte a lerna monorepos. Ele atravessa todos os pacotes dentro do monorepo e aplica cabealhos de copyright/licena.

        18. install-completion

        19. lb4 uninstall-completion
